class objects.Spring(stiffness, rest_length, index, type='linear', color='black', linewidth=2)[source]¶

Bases: object

Class for creating a Spring object. First the Spring itself is initialized. After that the initial conditions can be set. There are also Methods to compute the related force or the movement of the spring. A method for getting the values for each symbolic parameter as a dictionary is also implemented.

Parameters:

  • stiffness (int or float) – stiffness of the spring in (N/m)

  • rest_length (int or float) – the length of the spring in the rest position in (m)

  • index (int) – index of the spring in the system which is used for the Notation of the symbolic values

  • type (str) – the type of the spring (linear or cubic), default is “linear”

  • color (str) – color of the spring in the animation, default is “black”

  • linewidth (int or float) – the linewidth of the spring in the animation, default is 2

Variables:

  • stiffness (int or float) – stiffness of the spring in (N/m)

  • rest_length (int or float) – the length of the spring in the rest position in (m)

  • index (int) – index of the spring which is used for the Notation of the symbolic values

  • type (str) – type of the spring (linear or cubic)

  • color (str) – color of the spring in the animation

  • lw (int or float) – linewidth of the spring in the animation

  • sym_rest_length (sympy.Symbol) – symbolic rest length of the spring

  • sym_stiffness (sympy.Symbol) – symbolic stiffness of the spring

  • sym_Fx (sympy.Symbol) – symbolic force in x direction

  • sym_Fy (sympy.Symbol) – symbolic force in y direction

  • startingpoint (list) – startingpoint of the spring in (m)

  • endpoint (list) – endpoint of the spring in (m)

  • velocity (list of float) – starting velocity of the spring in (m/s)

  • length (float) – current length of the spring in (m)

  • prestretch (float) – prestretch of the spring in (m)

force(top_mass, bottom_mass)[source]¶

Method for computing the force of the spring in its current position. The force of the spring depends on the type of the spring. Also, it depends on the type of mass connected to the spring, because the steady body has a dimension and the mass point does not. If the inputs aren’t correct an assertion error should be raised.

Parameters:

  • top_mass (object or None) – the mass on which the spring is attached, if the created spring is the first spring in the system it is usually attached to the origin instead of a mass. Therefore, the value should be None in this case.

  • bottom_mass (object) – the mass which is connected with the endpoint of the spring

Returns:

forces of the spring in (N)

Return type:

list of sympy.Add or sympy.Mul

move(top_mass, bottom_mass)[source]¶

Method for computing the new start and end point of the spring depending on the positions of the masses. The current length is also computed. The computation depends on weather the connected mass is a masspoint or a steady body, because steady bodies have a dimension and masspoints do not

Parameters:

  • top_mass (object or None) – the mass on which the spring is attached, if the created spring is the first spring in the system it is usually attached to the origin instead of a mass. Therefore, the value should be None in this case.

  • bottom_mass (object) – the mass which is connected with the endpoint of the spring

Returns:

current startingpoint and endpoint of the spring in (m)

Return type:

list of lists of int or float

setInitialConditions(top_mass, bottom_mass, velocity)[source]¶

Method for setting the initial conditions of the spring. The user have the choice weather he wants to input a mass point or a steady body as a mass. Depending on the type of mass connected to the spring, the start and end point is set differently, because the steady body has a dimension and the mass point does not. If the inputs aren’t correct an assertion error should be raised.

Parameters:

  • top_mass (object or None) – the mass on which the spring is attached, if the created spring is the first spring in the system it is usually attached to the origin instead of a mass. Therefore, the value should be None in this case.

  • bottom_mass (object) – the mass which is connected with the endpoint of the spring

  • velocity (list) – the starting velocity of the spring in (m/s)
